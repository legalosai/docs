<Update label="July 24, 2026" description="Week of July 20 – July 24">

## New features

**L-1A petitions on platform.** LegalOS now supports L-1A (intracompany manager or executive) petitions end-to-end alongside the existing visa types.

- Full L-1A intake with a dedicated evidence criteria set seeded automatically when a petition is created.
- Auto-generation of the I-129L Supplement, mapped for L-1A filings.

**Multi-recipient filing sends and mark-as-filed.** File copies and filing documents can now be sent to more than one recipient, and petitions can be explicitly marked as filed once submitted to USCIS.

- Send file copies and receipt notices to multiple recipients on a single petition, tracked per recipient.
- A combined filing email bundles the filing documents into one send instead of sending them piecemeal.
- Mark a petition as filed to capture the filing date and reflect status downstream.

## Updates

- **Deep-linked "Incomplete Sections" in review.** The review step's Incomplete Sections list now links straight to the exact field that needs attention, so you can jump from the checklist to the missing input in one click instead of hunting through the section.

## Bug fixes

- **Account creation no longer misassigns company owners.** A server-side guard prevents new owner accounts from being silently attached to the wrong company when the client picker is skipped or misused — owners with no company selected are now routed to onboarding as intended.

</Update>

<Update label="July 20, 2026" description="Week of July 20">

## New features

**One-click recommendation letter drafts.** Generate a full recommendation letter draft for a recommender in a single click, grounded in their LinkedIn background so the tone and credentials line up with the person signing it.

- Recommender bios are enriched from LinkedIn before drafting so letters reference real accomplishments and titles.
- Business-letter format end-to-end — seeded prompts, agent instructions, and the DOCX export all produce a properly typeset business letter ready to send.
- A pre-generation URL gate confirms the recommender's LinkedIn is captured before a draft kicks off, so you don't burn a generation on missing context.

**Assistant workspace for letter drafting.** The letter-generation surface now has a working area built for iteration.

- Persistent quick actions stay in reach so you can steer a draft without hunting through menus.
- Stop or redraft on demand when a generation is heading the wrong direction.
- Clickable source references open the underlying evidence and deep-link into the exact spot they came from.
- A live draft timer shows generation progress in real time.

## Updates

- **Checkboxes now print on finalized signing copies.** The finalized, client-facing PDF for signing forms (including G-28 and I-129) now preserves every checked box when flattened for wet signature — checked boxes render as an X on the printed copy instead of disappearing. The working copy stays fully interactive and editable.

</Update>

<Update label="July 17, 2026" description="Week of July 13 – July 17">

## New features

**TN (USMCA professional) petitions.** LegalOS now supports TN petitions for Canadian and Mexican professionals under the USMCA, with two filing routes modeled as distinct visa types so you can pick the right path per case.

- **TN at the port of entry** for Canadian nationals — no I-129 required.
- **TN change of status / extension** through USCIS — files the I-129 plus the new Trade Agreement Supplement.
- Split beneficiary and employer intake with a dedicated profession/qualification step.
- Auto-generation of the I-129 Trade Agreement Supplement is wired into signing and signature-location flows alongside the base I-129.

**Online vs. mail USCIS filing fees.** The admin Filing Fee Invoice card now has a toggle for online vs. mail filing, and the seeded fee catalog carries both amounts where USCIS charges different rates. Switching modes re-prices only the catalog lines you haven't edited, so custom line items stay intact.

## Updates

- **Anvil form field auto-fit.** Long names, company names, and addresses now shrink to fit inside single-line fields on generated Anvil forms instead of clipping at the edges. Applies across every visa type.
- **Documents Needed checklist.** The documents-needed checklist now populates from bundle uploads before a section is marked complete, and bundle sections have their own Mark Complete action so progress reflects reality as you go.
- **Graceful version-skew handling on every route.** The update banner and stale-client refresh prompt are now mounted app-wide, so any route can surface a refresh prompt when a new build ships mid-session instead of silently failing a server action. Stale-client errors are also reported to Sentry.

## Bug fixes

- **Stable evidence strength ratings.** Per-bundle reported strength for critical role and original contributions no longer changes between views. Ratings are now cached against the evidence input and only re-evaluated when that input actually changes.
- **Download all evidence.** Fixed a crash in the admin "Download all evidence" action when an entry's source was stored as a provenance object rather than a plain string.

</Update>

<Update label="July 13, 2026" description="Week of July 13">

## New features

**E-3 petitions on platform.** You can now run E-3 (Australian specialty occupation) petitions end-to-end in LegalOS, alongside H-1B and other supported visa types.

- Full E-3 intake for both employer and beneficiary, including a dedicated current immigration status step, specialty occupation details, and processing options.
- Australian-nationality validation on the beneficiary side to catch eligibility issues before submission.
- Auto-generation of the E-3 form set (I-129, LCA, G-28, I-907) with E-3–specific I-129 mapping: classification set to E-3, basis 2.b, requested action 4.c, and Part 3 populated from the beneficiary's current status.
- Filings are routed to the Vermont Service Center (VSC) by default.
- No evidence module is required for E-3, so the workflow skips it automatically.

## Updates

- **Shared address component for H-1B secondary worksites.** The H-1B secondary work-location address now uses the same standardized address input as the rest of the platform, for a more consistent experience and cleaner data.

## Bug fixes

- **H-1B second worksite toggle now persists.** Unchecking "second physical work location" on an H-1B petition now correctly removes the secondary worksite instead of leaving stale data behind.

</Update>

<Update label="July 10, 2026" description="Week of July 6 – July 10">

## New features

**USCIS filing-fee Stripe invoicing.** Staff can now build itemized filing-fee invoices for a petition directly from the admin sidebar and send the client a single Stripe invoice with a hosted invoice page and pay link.

- Build a breakdown from the seeded USCIS filing-fee catalog via a dropdown, plus add editable custom line items.
- All amounts are staff-editable before sending, so you can reflect the current fee schedule or one-off adjustments per petition.
- The recipient signatory is resolved using the existing employer-signatory logic, and the Stripe customer's email stays in sync with the selected recipient.
- Sent (unpaid) invoices can be voided from the admin — the invoice is voided in Stripe and the local record moves to a `void` state.
- Send, finalize, and void calls are idempotent, so a double-click or retry can't create duplicate customers or invoices.

Filing-fee invoicing is gated on a Stripe API key. Until it's configured, the card shows a graceful "not configured" state and no invoicing actions run.

**Company Support Letter for every visa type.** The employer-signed Company Support Letter (previously labeled "H-1B Support Letter") is now available in the Form Signing company-signatory section for every visa type, including O-1A, EB-1A, and any future visa types — not just H-1B.

## Updates

- **Filing group in the admin sidebar.** The Filing Fee Invoice card lives in the sidebar's Filing group, and sidebar navigation now scrolls the target section into view below the sticky top nav.
- **Graceful release upgrades.** When a new version of LegalOS ships, the app no longer auto-reloads mid-session. Instead, an update banner appears so you can finish what you're doing before refreshing — a stale-page error will surface the banner instead of silently reloading and losing an in-progress form. Version checks now run on mount and when you refocus the tab, rather than polling every minute.
- **Additive auto-fill for visa history and passport countries.** Auto-fill for U.S. visa history and passport countries now merges into existing entries instead of overwriting them, so previously entered rows are preserved when new data is added.

## Coming soon

- Paid-status webhook and a client-facing invoice to-do so applicants can see and pay their filing-fee invoice from their portal.

</Update>
